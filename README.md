# auto-ria-telegram-bot

Telegram-бот, який моніторить нові оголошення на AUTO.RIA за фільтрами, які користувачі задають самі — через покроковий діалог у Telegram — і надсилає посилання, коли з'являється щось нове.

## Як це працює

Кожні 5 хвилин GitHub Actions запускає `npm run check`, який:

1. Забирає нові повідомлення в Telegram (`getUpdates`) від дозволених користувачів (`ALLOWED_CHAT_IDS`) і проводить їх через покроковий діалог створення фільтра (`src/commands.ts`), використовуючи AUTO.RIA API для розпізнавання марки/моделі.
2. Зберігає підписки в `data/subscriptions.json` — кожна підписка прив'язана до `chatId` того, хто її створив.
3. Для кожної підписки викликає AUTO.RIA API (`/auto/search`), порівнює з раніше побаченими ID (`data/seen.json`) і надсилає нові оголошення саме тому, хто створив фільтр.
4. Комітить оновлений стан (`data/`) назад у репозиторій.

При першому запуску нового фільтра повідомлення не надсилаються (щоб не заспамити всім наявним ринком) — лише запам'ятовуються поточні ID.

## Команди бота (Telegram)

- `/filter` — почати покроковий діалог створення нового фільтра (марка → модель → рік від → рік до → ціна від → ціна до → підтвердження). На будь-якому кроці можна написати `пропустити`, щоб не обмежувати за цим параметром.
- `/list` — показати свої фільтри з номерами.
- `/remove <номер>` — видалити фільтр за номером із `/list`.
- `/cancel` — скасувати поточний діалог створення фільтра.

Оскільки перевірка відбувається раз на ~5 хв, кожна відповідь бота в діалозі приходить із затримкою до 5 хв — це нормально, не збій.

Доступ обмежений списком `ALLOWED_CHAT_IDS` — усі інші отримають відповідь "У тебе немає доступу до цього бота."

## Налаштування

### 1. AUTO_RIA_API_KEY

Зареєструйся на https://developers.ria.com/, отримай особистий `api_key`.

### 2. Telegram-бот

1. Напиши [@BotFather](https://t.me/BotFather) → `/newbot` → отримаєш `TELEGRAM_BOT_TOKEN`.
2. Дізнайся `chat_id` кожного, кому потрібен доступ:
   - людина пише боту будь-яке повідомлення;
   - відкрий `https://api.telegram.org/bot<TOKEN>/getUpdates`;
   - знайди `"chat":{"id": ...}`.
3. Впиши всі дозволені `chat_id` через кому в `ALLOWED_CHAT_IDS`.

### 3. Локальний запуск

```bash
cp .env.example .env
# заповни .env своїми значеннями
npm install
npm run check
```

## Автоматичний запуск (безкоштовно, без сервера)

У репозиторії вже є `.github/workflows/check.yml`, який запускає перевірку кожні 5 хвилин через GitHub Actions.

1. Заливаєш цей проєкт у свій GitHub-репозиторій.
2. У `Settings → Secrets and variables → Actions` додаєш секрети:
   - `AUTO_RIA_API_KEY`
   - `TELEGRAM_BOT_TOKEN`
   - `ALLOWED_CHAT_IDS`
3. Workflow сам зберігатиме `data/` назад у репозиторій після кожного запуску (`permissions: contents: write` вже виставлено).
4. Можна запустити вручну через вкладку Actions → Check AUTO.RIA → Run workflow.

Мінімальний інтервал cron у GitHub Actions — 5 хв, але фактичний запуск може запізнюватись на кілька хвилин під навантаженням — це нормально.

## Структура даних (`data/`)

- `subscriptions.json` — список фільтрів користувачів.
- `seen.json` — вже показані ID оголошень, по одному масиву на кожен `id` підписки.
- `wizard_state.json` — незавершені діалоги створення фільтра (по одному на `chatId`).
- `telegram_offset.json` — позиція останнього обробленого повідомлення в Telegram, щоб не обробляти те саме двічі.

## Альтернатива без GitHub Actions

Той самий скрипт (`npm run check`) можна запускати через звичайний cron на власному ПК/VPS/Raspberry Pi — тоді `data/` просто залишається локальними файлами і GitHub Actions не потрібен. При такому варіанті інтервал можна зробити значно коротшим (наприклад, 30 сек), і діалог у Telegram відповідатиме майже миттєво.
